점수 계산 (Score)
calculate_v40_scores.py → ai_category_scores_v40, ai_final_scores_v40
Step 1. Rating 평균 = 카테고리 내 전체 Rating 합 / 개수   (범위: -4 ~ +4)
Step 2. Score = Rating 평균 × 2   (범위: -8 ~ +8)
Step 3. 카테고리 점수 = (Score × 0.5 + 6.0) × 10   (범위: 20 ~ 100점)
Step 4. 최종 점수 = 10개 카테고리 점수 합산   (범위: 200 ~ 1,000점)

예시: Rating 평균 2.77 → Score 5.54 → (5.54×0.5+6.0)×10 = 87.7 ≈ 88점

Bayesian Prior (베이지안 사전확률) 적용

공식 분해: 카테고리 점수 = (Rating Score × COEFFICIENT + PRIOR) × 10
상수값역할
PRIOR = 6.0 기본 점수 60점 베이지안 사전확률 (Bayesian Prior) — 평가 데이터가 없을 때 "보통(60점)"을 기본값으로 깔아주는 장치. 증거가 쌓일수록 60점에서 벗어나지만, 증거가 부족하면 60점 근처에 머무름
COEFFICIENT = 0.5 범위 스케일링 Rating Score(-8~+8)를 100점 만점 범위(20~100)에 맞추기 위한 스케일링 계수
PRIOR = 6.0 (베이지안 사전확률)을 왜 적용했는가?
  • 데이터 부족 시 안전장치 — AI가 X(제외) 판정을 많이 내려서 유효 평가가 2~3개뿐일 때, 기본값 없이 평균만 내면 극단적 점수가 나옴. Prior 60점이 "증거 부족 = 보통"이라는 합리적 출발점을 제공
  • 극단값 완화 — 평가가 한쪽(긍정 또는 부정)으로 쏠려도 Prior 60점이 중심 방향으로 자연스럽게 당겨줌
  • 신규 정치인 대응 — 활동 데이터가 적은 초선 정치인도 60점(보통)에서 시작하여 공정한 평가 가능
Prior의 핵심 의미

공식을 분해하면: 카테고리 점수 = Rating Score × 5 + 60
Prior 6.0이 만드는 60점은 단순한 숫자가 아니라 "선출직 공직자의 기본 신뢰"를 수치화한 것

상황 Rating Score 카테고리 점수 의미
전부 +4 (최고) +8 8×5+60 = 100점 증거가 Prior를 최대로 끌어올림
긍정/부정 균형 0 0×5+60 = 60점 증거가 Prior를 변경하지 못함 = 보통
전부 -4 (최저) -8 -8×5+60 = 20점 증거가 Prior를 최대로 끌어내림

Prior가 하는 일: "증거가 없으면 60점(보통)"이라는 출발점을 제공. 증거(Rating)가 쌓일수록 60점에서 위(최대 100) 또는 아래(최소 20)로 이동.
= 베이지안 통계의 핵심: 사전 믿음(Prior) + 관측 데이터(Rating) = 사후 믿음(점수)

부록 3: 베이지안 이론과 V40 평가 시스템

V40이 베이지안 이론의 어떤 요소를 적용했고, 어떤 요소를 적용하지 않았는지, 그리고 그 이유

1. 베이즈 정리 (Bayes' Theorem)

P(H|D) = P(D|H) × P(H) / P(D)
기호 이름 의미
P(H) Prior (사전 확률) 데이터를 보기 전의 믿음
P(D|H) Likelihood (우도) 가설이 참일 때 이 데이터가 나올 확률
P(H|D) Posterior (사후 확률) 데이터를 본 후 갱신된 믿음
P(D) Evidence (증거) 정규화 상수
핵심 구조: 사후 믿음 = 사전 믿음 × 관측 데이터의 영향
이것은 단순한 수학 공식이 아니라 "인간이 합리적으로 믿음을 갱신하는 방법"에 대한 이론

2. V40이 베이지안 이론에서 가져온 것

첫째, "사전 믿음(Prior)"이라는 개념

베이지안 이론의 핵심 주장: 데이터를 보기 전에도 합리적인 믿음이 존재함
빈도주의(Frequentist) 통계 — 오직 관측된 데이터만이 결론의 근거라고 봄. 사전 믿음을 인정하지 않음

V40은 베이지안의 손을 들어줌. 정치인 평가에서 데이터를 보기 전에도 이미 아는 것이 있음 — "이 사람은 선거를 통해 국민이 선택한 사람"
사전 지식을 무시하고 0에서 시작 = 빈도주의적 접근
사전 지식을 명시적으로 수치화하여 출발점으로 삼음 = 베이지안적 접근

Prior = 6.0 = "선출직 공직자에 대한 민주적 기본 신뢰"를 수치화한 것

둘째, "사전 믿음 + 데이터 = 갱신된 믿음"이라는 결합 구조

베이지안 이론에서 결론 = Prior 단독도, 데이터 단독도 아닌 둘의 결합
V40의 공식이 정확히 이 구조를 따름:
카테고리 점수 = Prior + 데이터 기여분
                      = 60   + Rating Score × 5
Prior만으로 점수가 결정되지 않음. 데이터만으로도 결정되지 않음.
둘이 합쳐져서 최종 판단이 됨 — 베이지안 이론의 기본 작동 원리이며, V40이 이 원리를 따름

셋째, "증거의 부재 ≠ 부재의 증거"라는 인식론

데이터가 아직 없는 정치인을 0점으로 처리하지 않음
"아직 모른다"와 "평가했는데 나쁘다"를 구분함 — 베이지안 인식론의 핵심 원칙

3. V40이 베이지안 이론에서 가져오지 않은 것, 그리고 그 이유

베이지안 갱신 메커니즘 — 데이터가 쌓이면 Prior가 약해지는 수학적 구조

정통 베이지안 시스템 — 데이터 축적될수록 Prior 영향 감소, 최종적으로 데이터가 결론을 지배함
V40에서 Prior는 고정. 데이터가 10개든 10,000개든 Prior 기여분은 항상 60점

가져오지 않은 이유: 정치 도메인의 본질

이유 1: 정치인이라는 존재의 전제

Prior 6.0이 인코딩하는 것 = "데이터 부족에 대한 보정"이 아닌 "선출직 공직자에 대한 기본 신뢰"
이 신뢰는 데이터가 쌓인다고 사라질 성질이 아님
국민이 선택했다는 사실은 평가 데이터가 1,000개 쌓여도 여전히 사실
법정의 무죄 추정이 증거가 많아져도 원칙으로서 유지되는 것과 동일

이유 2: 신인의 지속적 유입

매 선거마다 새로운 정치인이 대거 진입함
동적 계수 적용 시, 데이터 많은 중진과 데이터 적은 신인의 점수가 서로 다른 의미를 갖게 됨
같은 80점인데 하나는 데이터 기반, 하나는 Prior 기반이면 비교 자체가 불가능
고정 Prior = 모든 정치인에게 동일한 공식 적용 → 점수의 일관성과 비교 가능성 보장

4. V40의 정확한 위치

V40 = 베이지안 이론의 철학과 구조를 가져오되, 갱신 메커니즘은 정치 도메인의 특수성에 맞게 변형한 시스템
베이지안 이론 요소 적용 여부 근거
사전 믿음(Prior) 존재 적용 선출직의 민주적 기본 신뢰
Prior + Data = Posterior 결합 적용 60 + Rating Score × 5 = 카테고리 점수
증거 부재 ≠ 부재의 증거 적용 데이터 없는 정치인 ≠ 0점
데이터 축적 시 Prior 감소 미적용 민주적 신뢰는 소멸하지 않음 + 신인 지속 유입으로 공식 통일 필요
한 문장 요약: V40은 베이지안 이론에서 "합리적 사전 믿음을 명시하고, 이를 데이터와 결합하여 판단"하는 핵심 원리를 적용하되, Prior의 동적 감소는 정치 도메인의 본질 (선출직의 기본 신뢰, 신인의 지속적 유입)에 맞지 않으므로 고정 기준선으로 운용
