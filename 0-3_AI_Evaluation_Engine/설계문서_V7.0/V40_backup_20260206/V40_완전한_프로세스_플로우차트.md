# V40 완전한 프로세스 플로우차트

**작성일**: 2026-02-06
**버전**: V40
**목적**: 정치인 기본 정보 준비부터 상세평가보고서 생성까지 전체 프로세스 시각화

---

## 📊 전체 프로세스 개요

```
정치인 정보 작성 (MD)
    ↓
DB 등록 (politicians)
    ↓
데이터 수집 (Gemini 500 + Naver 500 = 1,000개)
    ↓
DB 저장 (collected_data_v40, rating 없음)
    ↓
검증 (URL, 도메인, 기간)
    ↓
AI 평가 (4 AIs × 1,000 = 4,000개, rating 부여)
    ↓
DB 저장 (evaluations_v40, collected_data_id 연결)
    ↓
점수 계산 (AI별 → 평균)
    ↓
DB 저장 (ai_final_scores_v40)
    ↓
보고서 생성 (4개 테이블 조인 → Markdown)
    ↓
파일 저장 (.md)
```

## 🎯 핵심 포인트 7가지

### 1️⃣ 세션 분리 = 객관성
```
수집 시점 (Gemini/Naver) ≠ 평가 시점 (4 AIs)
→ 타이밍 분리로 독립적 판단 보장
```

### 2️⃣ 풀링 방식
```
모든 AI가 전체 100개 평가 (독립적)
→ 편향 상쇄, 균형 잡힌 평가
```

### 3️⃣ 2단계 검증
```
수집 검증 (110% 이내) + 평가 검증 (97% 이상)
→ 품질 보장
```

### 4️⃣ DB 분리
```
수집 테이블 (rating 없음) ≠ 평가 테이블 (rating 있음)
→ collected_data_id로 연결
```

### 5️⃣ 점수 계산 4단계
```
Rating 평균 → 점수(×2) → 카테고리 점수 → 최종 점수
→ 일관된 계산 방식
```

### 6️⃣ 4개 테이블 조인
```
politicians + collected_data_v40 + evaluations_v40 + ai_final_scores_v40
→ 완전한 보고서 생성
```

### 7️⃣ 완전 무료
```
Gemini (무료) + Naver (무료) = $0/정치인
→ 비용 부담 없음
```

- 점수 단계: rating 평균 → 점수 변환
