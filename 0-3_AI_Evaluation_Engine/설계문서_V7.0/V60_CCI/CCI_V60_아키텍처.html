EVALUATION

Alpha 평가 — 플래툰 포메이션 (Claude 단독)

지휘관 (User) — 상호작용 최대 3회: ①임무부여 ②작전승인 ③해산승인
│
└── 소대장 (Opus, Teammate) — 평가 기준 통일 + 교차 검증 + 실시간 조율
│
├── 분대장 A (Sonnet, Teammate) — 후보 1: 정원오
│ ├── 분대원 1 (Haiku, Subagent) → A1-1 여론동향 평가
│ ├── 분대원 2 (Haiku, Subagent) → A1-2 이미지·내러티브 평가
│ ├── 분대원 3 (Haiku, Subagent) → A1-3 리스크 평가
│ ├── 분대원 4 (Haiku, Subagent) → A2-1 정당경쟁력 평가
│ ├── 분대원 5 (Haiku, Subagent) → A2-2 후보자경쟁력 평가
│ └── 분대원 6 (Haiku, Subagent) → A2-3 지역기반 평가
│
├── 분대장 B (Sonnet, Teammate) — 후보 2: 오세훈
│ └── (동일 구조, 6명 분대원)
│
├── 분대장 C (Sonnet, Teammate) — 후보 3: ...
│ └── (동일 구조)
│
└── 분대장 N (Sonnet, Teammate) — 후보 N: 경쟁자 수만큼 확장
└── (동일 구조)

소대장 ↔ 분대장: SendMessage (양방향, 실시간 조율)
분대장 → 분대원: Task tool (일회성, 결과만 반환)
AI: Claude 단독 (소대장 조율로 객관성 보장, 4AI 불필요)
비용: $0 (전원 Claude Code 구독 내)
조율 메커니즘

소대장의 3단계 조율

  • 사전 — 평가 기준 broadcast (동일 instruction 하달)
  • 중간 — 분대장 결과 수신 + 이상치 보정 요청
  • 사후 — 전 후보 결과 교차 검증 + 논리 일관성 확인
GPI vs ALPHA 비교

왜 Claude 단독으로 충분한가

  • GPI: 시민용 공개 → 4AI 다수결 (객관성 필수)
  • Alpha: 정치인 전용 비공개 → 소대장 조율 (구조적 보장)
  • V40: 조율자 없음 → AI 수로 보정
  • V60: 소대장 있음 → Claude만으로 OK
