## 🚨🚨🚨 CRITICAL: Alpha 평가 = 반드시 플래툰 포메이션 🚨🚨🚨

**V60 CCI Alpha 평가는 절대로 순차(1명씩) 실행을 금지한다.**
**반드시 플래툰 포메이션으로 병렬 실행해야 한다.**

### 이유

Alpha 평가는 **상대평가**다. 정원오의 +3과 오세훈의 +2가 의미를 가지려면,
**같은 시점, 같은 기준으로** 동시에 평가해야 한다.
순차로 한 명씩 하면 기준이 흔들릴 수 있다.

### 구조 (필수)

```
소대장 (team-lead, Opus, Teammate)
├── 사전 브로드캐스트: 모든 분대장에게 동일 평가 기준 하달
├── 교차 검증: 분대장 결과 비교·조율
│
├── Alpha 분대장 (Sonnet, Teammate) — 정치인 1
│   ├── 분대원 1 (Haiku, Subagent) — opinion  카테고리
│   ├── 분대원 2 (Haiku, Subagent) — media     카테고리
│   ├── 분대원 3 (Haiku, Subagent) — risk      카테고리
│   ├── 분대원 4 (Haiku, Subagent) — party     카테고리
│   ├── 분대원 5 (Haiku, Subagent) — candidate 카테고리
│   └── 분대원 6 (Haiku, Subagent) — regional  카테고리
│
├── Bravo 분대장 (Sonnet, Teammate) — 정치인 2
│   └── (분대원 6명, 동일 구조)
│
├── Charlie 분대장 (Sonnet, Teammate) — 정치인 3
│   └── (분대원 6명, 동일 구조)
│
└── Delta 분대장 (Sonnet, Teammate) — 정치인 4
    └── (분대원 6명, 동일 구조)
```

**1개 분대 = 1명 정치인**
**1명 분대원 = 1개 카테고리**
**모든 분대가 동시에 병렬 실행**

## 분대원 임무 (카테고리별)

각 분대원(Subagent, Haiku)은 아래를 수행:

```
1. fetch: python alpha_eval_helper.py fetch --politician_id={id} --politician_name="{name}" --category={cat}
2. JSON 파싱 → total_count 확인 (0이면 종료)
3. 25개씩 배치 분할
4. 배치별 평가 JSON 작성
5. save: python alpha_eval_helper.py save ... --input=eval_alpha_{cat}_batch_{N}.json
6. 임시 파일 삭제
7. 결과 분대장에게 반환
```

## 🚨🚨🚨 CRITICAL: 보고서 품질 균일성 조율 (소대장 필수 의무) 🚨🚨🚨

**2026-03-12 실전 교훈에서 도출된 규칙.**
**조은희 보고서 20KB/317줄 vs 정원오 39KB/759줄 — 같은 소대 산출물이 2배 격차 발생.**

### 근본 원인

소대장이 다음 3가지를 하지 않아 품질 불균형 발생:
1. 표준 출력 템플릿 미배포
2. 분대 결과물 교차 검증 미실시
3. 상대평가 특성상 필수인 분량·구조·깊이 균일성 기준 미하달

### 필수 규칙 (Phase 5 보고서 생성 시)

#### 규칙 1: 표준 구조 템플릿 사전 배포

소대장은 분대장에게 임무 지시 시 **반드시** 다음을 포함해야 한다:

```
보고서 표준 구조:
- 1장: 진단요약 (CCI 테이블 + 한줄진단 + 강점3 + 약점3 + 전략키워드)
- 2장: CCI 대시보드 (순위표 + 점수분해 + Alpha 카테고리별 비교표 with 그룹평균·우위)
- 3장: GPI 요약 (3축 분석 + 상하위 테이블 + 경쟁자 대비 10개 포지션표)
- 4장: Alpha 1 (opinion/media/risk 각각 점수표+경쟁자비교표+3단락분석+전략적함의)
- 5장: Alpha 2 (party/candidate/regional 동일 구조)
- 6장: 경쟁자 비교 (각 경쟁자별 16차원 비교표 + 공략포인트)
- 7장: SWOT (S/W/O/T 각 테이블 + SO/ST/WO/WT 교차전략 각 3개씩)
- 8장: 전략제언 (즉시실행+중기로드맵+3시나리오(조건+지표+전략+예상결과)+우선순위)
- 별첨1: GPI 참조
- 별첨2: Alpha 평가 데이터 개요
```

#### 규칙 2: 분량·구조 균일성 기준

```
최소 분량: 700줄 / 35KB 이상
최대 격차: 정치인 간 분량 차이 ±20% 이내
필수 테이블: 각 장에 명시된 테이블은 생략 금지
경쟁자 비교: 모든 정치인이 동일한 16차원 비교 구조 사용
SWOT 교차전략: 4유형(SO/ST/WO/WT) × 3개 = 최소 12개
```

#### 규칙 3: Phase 6.5 교차 검증 (소대장 필수 수행)

분대장 결과물 수집 후, 소대장은 **반드시** 다음을 교차 검증:

```
검증 항목:
1. 분량 비교: 각 보고서 줄 수/KB 비교 → ±20% 초과 시 재작성 지시
2. 구조 비교: 표준 구조 8장+별첨2 모두 존재하는지 체크리스트 확인
3. 테이블 비교: 각 장의 필수 테이블 존재 여부 확인
4. 깊이 비교: 각 카테고리 분석이 3단락 이상인지 확인
5. DB 수치 검증: 보고서 내 모든 점수가 DB 값과 일치하는지 확인

→ 1건이라도 미달이면 해당 분대장에게 보완 지시
→ 전체 균일성 확인 후에만 Phase 7(해산 승인) 진행
```

#### 규칙 4: 상대평가 조율 (Alpha 평가 단계)

Alpha 평가는 상대평가이므로, 소대장은 분대장 결과를 수집한 후:

```
1. 4명 정치인의 카테고리별 점수를 한 테이블에 정리
2. 상대적 순위가 논리적으로 타당한지 검증
3. 동일 사건에 대한 등급이 정치인 간 일관적인지 확인
4. 불일치 발견 시 해당 분대장에게 재평가 지시
```

### ⛔️ 절대 금지

```
❌ 분대장 결과물의 분량·구조를 비교하지 않고 바로 Phase 7 진행
❌ 한 정치인 보고서가 다른 정치인의 50% 미만 분량인데 통과
❌ 표준 구조의 장/절을 분대장 재량으로 생략 허용
❌ 교차 검증 없이 "내용이 정확하니까 OK" 판단
```
