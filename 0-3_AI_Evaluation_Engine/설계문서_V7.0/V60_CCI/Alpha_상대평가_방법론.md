# Alpha 상대평가 방법론 — 플래툰 포메이션 기반

**작성일**: 2026-03-11

## 핵심 결론

### 1. 레이팅 체계: GPI와 동일한 +4 ~ -4

Alpha 데이터도 GPI와 같은 +4~-4 등급 체계로 평가한다.

```
+4(+8점) 탁월 — 여론 압도적 1위, 정당 지지율 독보적
+3(+6점) 우수 — 안정적 선두, 현직 재선 우위
+2(+4점) 양호 — 긍정적 여론, 유리한 구조적 조건
+1(+2점) 보통 — 기본적 경쟁력, 평균적 상황
-1(-2점) 미흡 — 하락세 여론, 구조적 약점
-2(-4점) 부족 — 부정 여론 우세, 불리한 경쟁 구도
-3(-6점) 심각 — 치명적 리스크, 대규모 이탈
-4(-8점) 최악 — 당선 가능성 사실상 없음
X       제외 — 무관한 데이터
```

### 2. 별도 상대평가 알고리즘 불필요

**레이팅 자체가 이미 상대평가를 반영한다.**

예시:
- AI가 "정원오 1위 37.8%"를 읽으면 → 경쟁 맥락을 이해하고 +4를 매김
- AI가 "오세훈 23.7% 2위"를 읽으면 → 1위 대비 열세를 이해하고 +2~+3을 매김
- 같은 여론조사 기사라도 후보별 위치에 따라 자연스럽게 다른 등급

→ Elo, Z-Score, TOPSIS 등 별도 상대평가 알고리즘 불필요

### 3. 플래툰 포메이션 = 상대평가 일관성 보장 구조

상대평가가 성립하려면 **같은 기준으로** 모든 후보를 평가해야 한다.
플래툰 포메이션의 3계층 구조가 이를 보장한다.

## 플래툰 포메이션 3계층 구조

```
소대장 (Opus, Teammate)
├── 평가 기준 통일: 모든 분대장에게 동일한 평가 instruction 하달
├── 교차 검증: 분대장들의 결과를 비교·조율
├── 조율: SendMessage로 실시간 양방향 통신
│
├── 분대장 A (Sonnet, Teammate) — 후보 1: 정원오
│   ├── 분대원 1 (Haiku, Subagent) — 여론동향 평가
│   ├── 분대원 2 (Haiku, Subagent) — 이미지·내러티브 평가
│   ├── 분대원 3 (Haiku, Subagent) — 리스크 평가
│   ├── 분대원 4 (Haiku, Subagent) — 정당경쟁력 평가
│   ├── 분대원 5 (Haiku, Subagent) — 후보자경쟁력 평가
│   └── 분대원 6 (Haiku, Subagent) — 지역기반 평가
│
├── 분대장 B (Sonnet, Teammate) — 후보 2: 오세훈
│   └── (동일 구조, 6명 분대원)
│
├── 분대장 C (Sonnet, Teammate) — 후보 3: ...
│   └── (동일 구조)
│
└── ... (경쟁 후보 수만큼 분대)
```

### Teammate vs Subagent 구분 (CRITICAL!)

| 역할 | 구현 방식 | 통신 | 수명 | 조율 가능 |
|------|----------|------|------|----------|
| **소대장** | Teammate (SendMessage) | 양방향 | 상주 | ✅ 가능 |
| **분대장** | Teammate (SendMessage) | 양방향 | 상주 | ✅ 가능 |
| **분대원** | Subagent (Task tool) | 결과만 반환 | 일회성 | ❌ 불가 |

**핵심**: 분대장은 Teammate이므로 소대장과 **실시간 양방향 통신**이 가능하다.
이것이 상대평가 일관성을 보장하는 구조적 근거다.

## 왜 Claude 단독으로 충분한가?

### GPI vs CCI Alpha 비교

| 항목 | GPI (V40) | CCI Alpha |
|------|-----------|-----------|
| **대상** | 시민용 (공개) | 정치인 전용 (비공개) |
| **핵심 가치** | **객관성이 생명** | 전략적 분석이 핵심 |
| **비판 방어** | "편향 AI 아니냐" 비판 대응 필요 | 해당 정치인만 보므로 부담 낮음 |
| **평가 AI** | 4개 AI 다수결 (Claude+ChatGPT+Gemini+Grok) | **Claude 단독** |
| **조율자** | 없음 → 다수결로 편향 보정 | **소대장 (Opus)** → 통일된 기준 |
| **일관성 보장** | AI 수로 보정 | 플래툰 포메이션 구조로 보장 |

### V40이 4개 AI가 필요했던 이유

V40은 **조율자가 없다.** 각 AI가 독립적으로 평가하고, 결과를 다수결로 합산한다.
→ 개별 AI의 편향을 **AI 수(4개)**로 상쇄

### CCI Alpha가 Claude 단독으로 충분한 이유

CCI Alpha는 **소대장이 조율한다.**
- 소대장이 모든 분대장에게 **동일한 평가 기준** 하달
- 분대장들이 평가한 결과를 소대장이 **교차 검증** 및 **조율**
- SendMessage 양방향 통신으로 **실시간 보정** 가능
→ **구조적으로** 일관성이 보장되므로 다수 AI 불필요

## 소대장의 조율 메커니즘

### 1. 사전 조율 (평가 전)

```
소대장 → 전체 분대장에게 broadcast:
  "Alpha 1 여론동향 평가 기준:
   - 여론조사 1위 격차 10%p+ → +4
   - 1위 격차 5-10%p → +3
   - 1위 격차 0-5%p → +2
   - 2-3위권 → +1
   - 하위권 → -1 ~ -4
   이 기준으로 통일하세요."
```

### 2. 중간 조율 (평가 중)

```
분대장 A → 소대장: "정원오 여론동향 평균 +3.2입니다"
분대장 B → 소대장: "오세훈 여론동향 평균 +2.8입니다"
소대장 → 분대장 B: "오세훈이 23.7%인데 +2.8이면 높지 않나요?
                    정원오 37.8% 대비 14%p 격차입니다. 재검토 요청."
```

### 3. 사후 검증 (평가 후)

```
소대장: 전 분대장 결과 종합
  - 정원오: Alpha 1 = +3.2 / Alpha 2 = +2.5
  - 오세훈: Alpha 1 = +2.1 / Alpha 2 = +3.0
  → 정원오 여론 우위, 오세훈 구조 우위 → 논리적으로 타당 ✅
```

## 6개 카테고리별 분대원 배치

각 분대장은 6명의 분대원(Subagent)을 Task tool로 생성하여 카테고리별 평가를 수행한다.

| # | 카테고리 | Alpha | 분대원 역할 |
|---|---------|-------|-----------|
| 1 | 여론동향 | Alpha 1 | 여론조사 추이 + 바람 효과 데이터 평가 |
| 2 | 이미지·내러티브 | Alpha 1 | 미디어 논조 + SNS + 브랜드 일관성 데이터 평가 |
| 3 | 리스크 | Alpha 1 | 네거티브 취약점 데이터 평가 |
| 4 | 정당경쟁력 | Alpha 2 | 정당 지지율 + PVI 데이터 평가 |
| 5 | 후보자경쟁력 | Alpha 2 | 현직효과 + 인지도 + 공천 상태 데이터 평가 |
| 6 | 지역기반 | Alpha 2 | 연고 + 조직동원력 + 지역 서비스 데이터 평가 |

**분대원은 Subagent(Task tool)이므로:**
- 결과만 분대장에게 반환
- 양방향 통신 불가
- 일회성 (평가 완료 후 소멸)
- 분대장이 결과를 종합하여 소대장에게 보고
