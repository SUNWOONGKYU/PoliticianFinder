## 2. 톤 앤 매너 가이드

### 기본 원칙 3가지

**원칙 1: 중립적 사실 서술**
- 편향 없는 표현 사용
- "좋다/나쁘다" 대신 데이터 제시
- 확정된 사실만 단정적 표현

**원칙 2: 데이터 기반 서술**
- 모든 주장에 근거 명시
- 출처 표기 필수
- 통계와 수치 활용

**원칙 3: 논란은 객관적으로**
- 확정 vs 의혹 명확히 구분
- 양측 입장 균형있게 서술
- 진행 중 사안은 현재 상태 표기

---

### 좋은 표현 예시 (10개)

**1. 데이터 중심 표현**
```
✅ "ChatGPT 평가 881점으로 4개 AI 중 최고점"
✅ "조은희 의원은 leadership 카테고리에서 평균 대비 +12점 우위"
✅ "시민 평가 4.2/5.0점 (응답자 1,247명)"
```

**2. 중립적 논란 서술**
```
✅ "2022년 3월 재산 신고 누락 의혹으로 특검 조사 진행 중"
✅ "A 기업 후원금 수령 논란 (2023.05.12 기사)"
✅ "공약 이행률 65% (17개 중 11개 완료, 6개 진행 중)"
```

**3. 객관적 평가 인용**
```
✅ "Claude 평가 근거: '국정감사 우수의원 선정으로 전문성 입증'"
✅ "Gemini +3점 평가: '재난 대응 신속성 인정'"
✅ "AI 4개 모두 +2점 이상 (합의 평가)"
```

**4. 확정 vs 의혹 구분**
```
✅ "2021년 정치자금법 위반으로 벌금 500만 원 확정 (대법원 판결)"
✅ "이해충돌 의혹 제기됨 (검찰 수사 중, 결론 미확정)"
```

**5. 통계 표현**
```
✅ "AI 평가 표준편차 8.9점 (평가 일관성 높음)"
✅ "긍정 평가 75.0% (ChatGPT), 65.4% (Claude)"
✅ "상위 15% 점수 (전체 정치인 평균 750점 대비 +66점)"
```

**6. 비교 표현**
```
✅ "타 정치인 평균 대비 leadership +12점, integrity -5점"
✅ "같은 정당 내 순위 3위 (총 18명 중)"
✅ "전국 기초단체장 평균 782점 대비 +34점"
```

**7. 시간 범위 명시**
```
✅ "2016년 이후 활동 기준 (최근 8년)"
✅ "2023년 7월~2024년 6월 데이터"
✅ "현재 진행 중 (2024년 9월 기준)"
```

**8. 긍정 사례**
```
✅ "2023년 6월 스마트시티 조례 제정으로 시민 편의 증대 (ChatGPT +3)"
✅ "재난 안전망 구축으로 응급 대응 시간 30% 단축 (Grok +4)"
```

**9. 부정 사례**
```
✅ "2022년 예산 낭비 지적 (감사원 보고서, Gemini -2)"
✅ "공약 지연으로 시민 불만 제기 (Claude -3)"
```

**10. AI 평가 종합**
```
✅ "AI 4개 중 3개가 +3 이상 (ChatGPT +3, Grok +3, Gemini +3, Claude +2)"
✅ "평가 편차 1.27점 (일관성 매우 높음)"
```

### 중립적 서술 가이드라인

**논란 서술 시 4단계 체크리스트:**

- [ ] **확정 사실인가, 의혹인가?**
  - 확정: "~로 확정" (판결, 조사 결과)
  - 의혹: "~의혹 제기" (수사 중, 미확정)

- [ ] **출처가 명확한가?**
  - 법원 판결문, 감사원 보고서, 언론 보도 등

- [ ] **시간 정보가 정확한가?**
  - 날짜, 기간 명시

- [ ] **양측 입장이 있는가?**
  - 논란 사안은 양측 입장 모두 서술

**예시:**
```
✅ "2022년 3월 재산 신고 누락 의혹 제기 (A일보 보도)
    - 의혹: 재산 5억 원 미신고
    - 본인 해명: '배우자 명의로 신고 완료'
    - 현재 상태: 선관위 조사 진행 중 (2024.09 기준)
    - AI 평가: Claude -3, Gemini -2 (윤리성 카테고리)"
```

## 3. 섹션별 상세 가이드

### 섹션 1: 정치인 프로필 (~30줄)

#### 목적
- 보고서를 읽는 사람이 정치인을 빠르게 이해할 수 있도록 기본 정보 제공
- 정치인 본인이 참고할 수 있는 요약된 프로필

#### 구성 요소

**1.1 기본 정보**
```markdown
**이름**: {이름}
**소속**: {정당}
**현재 직책**: {직책}
```

**1.2 주요 경력**
```markdown
- **{직책}** ({기간})
- **{직책}** ({기간})
...

**총 {분야} 경력**: {년}년
```

**1.3 전문 분야**
```markdown
- **{분야1}**: {설명}
- **{분야2}**: {설명}
- **{분야3}**: {설명}
```

**1.4 정치적 특징**
```markdown
- **{특징1}**: {사례}
- **{특징2}**: {사례}
- **{특징3}**: {사례}
```

#### 톤
- 중립적, 사실 중심
- 긍정/부정 평가 배제
- 나열식 서술

#### 필수 요소
- 이름, 현재 직책, 정당, 주요 경력 3개
- 전문 분야 3개 이내
- 정치적 특징 3개 이내

#### 금지 사항
- 주관적 평가 ("훌륭한", "우수한" 등)
- 미래 예측 ("~할 것")
- 애매모호한 표현 ("어느 정도", "상당히" 등)

#### 예시 문장
```
✅ "조은희, 1968년생, 국민의힘 소속 양천구청장 (2018년~현재)"
✅ "주요 경력: 서울시의원 3선 (2006~2018), 국회의원 보좌관 (2002~2006)"
✅ "전문 분야: 도시 재생, 복지 정책, 교육 개혁"
```

#### 작성 원칙
- 객관적 사실만 (검증 가능한 정보)
- 최신 정보 우선 (현재 직책 강조)
- 주관적 평가 금지 (이 섹션에서는)

### 섹션 2: 평가 요약 (~90줄)

#### 목적
- 보고서 전체를 **3분 안에 파악**할 수 있도록 핵심 요약
- 기존 Executive Summary + 평가 요약을 **하나로 통합**
- 정치인이 자신의 평가를 빠르게 이해

#### 구성 요소

**2.1 최종 점수 및 등급**
```markdown
**최종 점수**: **{점수}점** / 1,000점
**등급**: **{등급} ({등급명})**
```

**2.2 한 줄 평가 (건설적 제안형)**
```markdown
> **"{강점 카테고리} 분야에서 높은 AI 합의를 얻었으며, {약점 카테고리} 강화 시 종합 평가 상승 여지가 큼"**
```

**구조**: `[상위 카테고리 강점 합의] + [하위 카테고리 개선 가능성]`
- 강점 선택: 카테고리 점수 상위 3개 중 대표
- 약점 선택: 카테고리 점수 하위 3개 중 대표

**예시:**
```
✅ "비전·전문성 분야에서 높은 AI 합의를 얻었으며, 청렴성 강화 시 종합 평가 상승 여지가 큼"
✅ "전문성·소통 분야의 강점이 뚜렷하며, 책임감 보완 시 상위 등급 진입 가능"
```

**2.3 핵심 인사이트 (V40.2 추가)**
```markdown
- {인사이트 1: 4개 AI 모두 합의한 강점/약점}
- {인사이트 2: AI 간 편차가 큰 카테고리 + 의미}
- {인사이트 3: 데이터 신뢰도 관련 핵심 사항}
```

**2.4 AI별 점수**
```markdown
| AI | 점수 |
|---|:---:|
| ChatGPT | {점수}점 |
| Grok | {점수}점 |
| Gemini | {점수}점 |
| Claude | {점수}점 |
| **4 AIs 평균** | **{점수}점** |
```

**2.5 카테고리별 점수 표 (10개)**
```markdown
| 카테고리 | 점수 | 평가 |
|---------|:----:|:----:|
| {카테고리1} | {점수}점 | ⭐ 최고 |
| {카테고리2} | {점수}점 | ⭐ |
...
| {카테고리10} | {점수}점 | ⚠️ 개선 필요 |
```

**2.6 긍정/부정/X 비율**
```markdown
긍정 평가 데이터: ████████████████████ {%} ({개수}개)
부정 평가 데이터: █ {%} ({개수}개)
평가 제외 (X):    ██ {%} ({개수}개)

⚠️ 참고: 이것은 AI가 수집한 뉴스/자료 데이터의 비율이며,
시민 여론조사 결과가 아닙니다.
```

#### 톤
- 간결, 명확
- 숫자 중심
- 건설적 제안형 한 줄 평가

#### 필수 요소
- 최종 점수, AI별 점수, 한 줄 평가
- 핵심 인사이트 3~4줄 (V40.2)
- 카테고리 점수 표
- 긍정/부정/X 비율

#### 작성 원칙
- 숫자 중심 (직관적 이해)
- 시각적 표현 (표, 막대그래프)
- 명확한 메시지 (한 줄 평가)
- 건설적 방향 제시 (개선 가능성 강조)

---

### 섹션 3: 강점 분석 TOP 3~5 (~180줄) ★주력

#### 목적
- **카테고리 점수 상위 3~5개**를 선정하여 왜 강점인지 분석
- 실행 가능한 **강화 방향** 제시
- 정치인이 강점을 어떻게 더 강화할지 구체적 가이드 제공

#### 핵심 원칙: 점수 기반 분석

```
⚠️ 절대 금지: 개별 뉴스 사례 나열, 특정 뉴스 제목 인용
✅ 올바른 방식: 카테고리 점수, AI 일치도, 긍정/부정 비율 등 통계 기반 분석
```

#### 강점 선정 기준
1. **4 AIs 평균 점수** 상위 3~5개 카테고리
2. AI 4개 모두 80점 이상인 카테고리 우선
3. 동점 시 AI 간 편차가 작은(일치도 높은) 카테고리 우선

#### 구조 (강점 1개당 약 36~45줄)

```markdown
### 강점 {N}: {카테고리명} ({점수}점) ⭐

#### 왜 강점인가
- 4개 AI 평균 {점수}점, 10개 카테고리 중 {N}위
- AI별 점수: ChatGPT {점수}점, Grok {점수}점, Gemini {점수}점, Claude {점수}점

#### AI 일치도
- 표준편차 {N}점 ({해석: "4개 AI 모두 80점 이상 → 높은 합의" 등})
- 가장 높은 AI: {AI명} ({점수}점)
- 가장 낮은 AI: {AI명} ({점수}점)
- 차이: {N}점

#### 긍정/부정 비율
- 긍정 {%}%, 부정 {%}%, X {%}%
- {해석: "부정 비율이 매우 낮아 논란 없는 강점" 등}

#### 핵심 강점 요인 (V40.2: 3~5문장 패턴 서술)
{3~5문장 패턴 서술. 특정 뉴스 제목이 아닌, 이 카테고리가 왜 높은지 패턴 분석}
{예: 정책 발표 빈도, 의정활동 일관성, 긍정 평가의 공통 패턴 등을 서술}
{AI 평가 근거의 공통 방향성을 종합하되, 개별 뉴스 인용은 금지}

#### 강화 방향 ⭐

1. **{전략1 제목}**

## 7. ChatGPT 인용 규칙

### 기본 원칙

ChatGPT가 제시한 예측이나 추측을 보고서에 포함할 때는 **반드시 ChatGPT 출처를 명확히** 해야 합니다.

### ❌ 잘못된 예시 (ChatGPT 추측을 보고서 주장처럼 표현)

```
❌ "이러한 활동을 통해 향후 시민 만족도가 더욱 높아질 것으로 기대됩니다."
   → 누가 기대하는가? 보고서 작성자? ChatGPT? 불명확!

❌ "리더십이 계속 발휘될 것입니다."
   → 미래 예측의 근거가 없음

❌ "앞으로도 청렴도가 유지될 전망입니다."
   → 누가 전망하는가? 출처 불명
```

### ✅ 올바른 예시 (ChatGPT 출처 명확히)

```
✅ "ChatGPT는 '이러한 활동을 통해 향후 시민 만족도가 더욱 높아질 것'으로 예측했습니다."
   → ChatGPT가 예측했다고 명확히 표기

✅ "ChatGPT 평가 근거: '리더십이 계속 발휘될 것으로 기대됨'"
   → 평가 근거 인용문으로 표시

✅ "ChatGPT는 '앞으로도 청렴도가 유지될 것'으로 평가"
   → AI 평가로 명확히 구분
```

### 인용 형식

**형식 1: 직접 인용**
```
ChatGPT는 "[원문 그대로]"라고 평가했습니다.
```

**형식 2: 평가 근거 인용**
```
ChatGPT 평가 근거: "[원문 그대로]"
```

**형식 3: 예측 표기**
```
ChatGPT는 ~할 것으로 예측/전망/기대
```

### 중요 체크포인트

- [ ] 미래 예측 표현이 있는가?
- [ ] 그것이 ChatGPT 평가에서 온 것인가?
- [ ] ChatGPT 출처를 명확히 표기했는가?
- [ ] 보고서 자체 주장과 구분되는가?

### 예외: 과거 사실 기반 표현

```
✅ "2023년 스마트시티 조례 제정 이후 범죄율이 20% 감소했습니다."
   → 과거 사실, ChatGPT 인용 불필요

✅ "2024년 9월 기준, 공약 이행률 65%를 기록하고 있습니다."
   → 현재 사실, ChatGPT 인용 불필요

❌ "이러한 추세가 계속될 것입니다."
   → 미래 예측, ChatGPT 출처 필요
```

## 8. 전체 작성 가이드라인

### 필수 원칙 10가지

1. **객관성**: 편향 없는 중립적 서술
2. **근거 중심**: 모든 주장에 데이터 기반 (점수, 비율, 편차)
3. **구조화**: 섹션별 일관된 형식
4. **시각화**: 표, 그래프 적극 활용
5. **중복 제거**: 같은 데이터/사례가 여러 섹션에 반복되지 않도록
6. **실행 가능성**: 실질적 개선/강화 방안 제시
7. **투명성**: 한계와 유의사항 명확히 고지
8. **정확성**: 숫자, 날짜, 출처 정확히
9. **가독성**: 긴 문장 지양, 명확한 표현
10. **유용성**: 정치인이 실제로 활용 가능하도록

## 9. 데이터 소스 위치

### 평가 데이터 저장 위치

보고서 작성 시 참조하는 원본 데이터는 다음 위치에 저장되어 있습니다:

```

**예시: 조은희 의원 (Gemini 평가)**
```
results/evaluate/gemini/조은희/
├── expertise_수집.json      (Gemini가 수집한 원본 자료)
├── expertise_평가.json      (Gemini의 평가 결과)
├── leadership_수집.json
├── leadership_평가.json
├── vision_수집.json
├── vision_평가.json
... (10개 카테고리 × 2 = 20개 파일)
```

### 파일 구조

**{카테고리}_수집.json**:
- Gemini가 수집한 원본 자료 (뉴스, 공식 기록 등)
- 형식: `[{"title": "...", "content": "...", "source": "...", "source_url": "...", "date": "..."}]`

**{카테고리}_평가.json**:
- Gemini의 평가 결과 (rating, score, rationale)
- 형식: `{"evaluations": [{"id": "...", "rating": "+3", "score": 6, "rationale": "..."}]}`

1. expertise (전문성)
2. leadership (리더십)
3. vision (비전)
4. integrity (청렴성)

7. transparency (투명성)

### 중요 사항

- 반드시 해당 정치인의 폴더에서 데이터를 가져올 것
