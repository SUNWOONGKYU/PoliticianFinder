# Perplexity가 OFFICIAL 데이터 수집을 어려워하는 이유

**작성일**: 2026-01-20
**발견**: 사용자 테스트 결과, Perplexity는 OFFICIAL 데이터 수집에 약함
**목적**: 원인 분석 및 해결 방안 제시

---

## 📊 발견사항 요약

```
사용자 테스트 결과:
❌ Perplexity는 OFFICIAL 데이터 (.go.kr, assembly.go.kr) 수집이 어려움
✅ Gemini는 OFFICIAL 데이터 수집이 원활함

결론: 50-50 분담 불가능, 75-25 또는 역할 분리 필요
```

## 🔍 7가지 핵심 이유

### 이유 1: 검색 엔진 인덱싱 차이 (가장 큰 이유)

#### 문제 구조

```
Gemini:
└── Google Search 직접 사용
    ├── Google의 한국 서버 활용
    ├── 한국 사이트 인덱싱 강력
    └── .go.kr 도메인 우선 순위 높음

Perplexity:
└── 자체 검색엔진 (독립 크롤링)
    ├── 독립적인 웹 인덱싱
    ├── 한국 사이트 인덱싱 약함
    └── .go.kr 도메인 우선 순위 낮음
```

#### 구체적 예시

```
검색어: "조은희 의원 법안"

Gemini (Google Search):
✅ assembly.go.kr - 국회 의안정보
✅ likms.assembly.go.kr - 의안 상세
✅ mois.go.kr - 행안부 보도자료
✅ korea.kr - 정부 공식 자료

Perplexity (자체 검색엔진):
⚠️ 뉴스 기사 (조선, 중앙, 한겨레)
⚠️ 위키피디아
⚠️ 블로그
❌ .go.kr 사이트 거의 없음
```

#### 왜 이런 차이?

```
Google Korea의 강점:
1. 한국에 데이터센터 있음
2. 한국 사이트 크롤링 주기 짧음
3. 한국어 NLP 최적화
4. .go.kr 도메인 중요도 학습됨

Bing/Others의 약점:
1. 글로벌 서버 (한국 최적화 부족)
2. 한국 사이트 크롤링 주기 길음
3. 한국어 처리 약함
4. .go.kr 도메인 학습 부족

Perplexity:
→ 여러 검색 엔진 혼합하지만
→ 약한 엔진(Bing 등)이 포함되어
→ 전체 성능이 평균으로 떨어짐
```

### 이유 2: JavaScript 렌더링 문제

#### 정부 사이트의 기술 스택

```
많은 정부 사이트는 최신 프레임워크 사용:
- React
- Vue.js
- Angular

문제:
└── CSR (Client-Side Rendering)
    ├── 초기 HTML: <div id="app"></div> (빈 껍데기)
    ├── JavaScript 실행 후: 실제 콘텐츠 렌더링
    └── 크롤러가 JS 미실행 시: 빈 페이지 인식
```

#### 예시: 국회 의안정보시스템

```html
<!-- 초기 HTML (크롤러가 보는 것) -->
<!DOCTYPE html>
<html>
<head>...</head>
<body>
  <div id="app"></div>
  <script src="bundle.js"></script>
</body>
</html>

→ 크롤러: "아무 내용 없네?" ❌

<!-- JavaScript 실행 후 (브라우저에서 보는 것) -->
<div id="app">
  <h1>의안 정보</h1>
  <p>법안명: ...</p>
  <p>발의자: 조은희</p>
  ...
</div>

→ 사람: "내용 있네!" ✅
```

#### 검색 엔진별 JavaScript 렌더링 지원

| 검색 엔진 | JS 렌더링 | 품질 | 비고 |
|----------|----------|------|------|
| **Google** | ✅ 지원 | ⭐⭐⭐⭐⭐ | 매우 강력 |
| Bing | ⚠️ 제한적 | ⭐⭐⭐ | 일부만 지원 |
| DuckDuckGo | ❌ 미지원 | ⭐ | 기본 HTML만 |
| Others | ❌ 대부분 미지원 | ⭐ | 기본 HTML만 |

```
결과:
- Gemini (Google 사용): JavaScript 렌더링 완벽 지원
- Perplexity (자체 엔진): 일부만 지원 → 콘텐츠 누락
```

### 이유 3: 로그인/세션 요구

#### 문제

```
일부 정부 사이트 특징:
1. 회원 가입/로그인 필요
2. 쿠키/세션 필요
3. CAPTCHA 있음
4. IP 기반 접근 제한

검색 엔진 크롤러:
❌ 로그인 불가능
❌ 세션 유지 불가능
❌ CAPTCHA 통과 불가능
❌ IP 제한 우회 불가능

결과:
→ 크롤러가 콘텐츠 접근 못 함
→ 인덱싱 안 됨
→ 검색 결과에 안 나옴
```

#### 예시

```
국회 의안 상세 페이지:
https://likms.assembly.go.kr/bill/detail.do?billId=XXX

접근 시:
1. 쿠키 확인
2. 없으면 → 로그인 페이지로 리다이렉트
3. 크롤러: "로그인 페이지만 보임" ❌

공공데이터포털:
https://www.data.go.kr/...

접근 시:
1. API 키 필요
2. 회원만 상세 정보 열람
3. 크롤러: "회원 가입 안내만 보임" ❌
```

### 이유 4: SEO 최적화 부족

#### 정부 사이트의 SEO 특징

```
SEO (Search Engine Optimization) 요소:

민간 웹사이트 (뉴스, 블로그):
✅ <title> 최적화
✅ <meta description> 상세
✅ <h1>, <h2> 구조화
✅ alt 텍스트 완벽
✅ sitemap.xml 제공
✅ robots.txt 적절
✅ 내부 링크 풍부
✅ 외부 링크 많음 (backlinks)

정부 사이트:
❌ <title>: "홈페이지" (의미 없음)
❌ <meta description>: 없음
❌ 구조화 엉망
❌ alt 텍스트 없음
❌ sitemap 없음
❌ robots.txt로 크롤링 제한
❌ 내부 링크 부족
❌ 외부 링크 거의 없음
```

#### 예시

```html
<!-- 좋은 SEO 예시 (뉴스 기사) -->
<head>
  <title>조은희 의원, 지방자치법 개정안 발의...주요 내용은?</title>
  <meta name="description" content="조은희 의원이 지방자치단체의 재정 자율성 강화를 위한 지방자치법 개정안을 발의했다. 주요 내용은...">
  <meta name="keywords" content="조은희, 지방자치법, 개정안, 국회">
</head>

→ 검색 엔진: "오! 좋은 콘텐츠네!" ✅

<!-- 나쁜 SEO 예시 (정부 사이트) -->
<head>
  <title>국회 의안정보시스템</title>
  <meta name="description" content="">
  <meta name="keywords" content="">
</head>

→ 검색 엔진: "이게 뭔 내용인지 모르겠네?" ❌
```

#### 왜 정부 사이트는 SEO를 안 할까?

```
민간 웹사이트:
- 트래픽 = 수익 (광고, 구독)
- SEO 투자 필수
- SEO 전문가 고용

정부 사이트:
- 트래픽 필요 없음 (공공 서비스)
- SEO 예산 없음
- SEO 중요성 인식 부족
- "찾아올 사람은 알아서 찾아온다" 마인드

결과:
→ 검색 엔진 랭킹 낮음
→ 검색 결과 후순위
→ Perplexity 같은 AI는 못 찾음
```

```
Perplexity AI:

- 글로벌 검색 엔진 사용

- .go.kr 도메인 (한국 전용 TLD)

결과:
→ Perplexity의 검색 알고리즘이
→ 한국 정부 사이트 중요도를 낮게 평가
```

#### 비교

```
영어권 정부 사이트 (.gov):
- 글로벌 접근성 높음
- 검색 엔진 인덱싱 잘 됨
- Perplexity도 잘 찾음

"Biden executive order" 검색:
✅ whitehouse.gov
✅ congress.gov
✅ regulations.gov
→ Perplexity가 잘 찾음

한국 정부 사이트 (.go.kr):
- 글로벌 접근성 낮음
- 검색 엔진 인덱싱 약함
- Perplexity가 못 찾음

예시:
"조은희 의원 법안" 검색:
❌ assembly.go.kr
❌ likms.assembly.go.kr
→ Perplexity가 못 찾음
```

### 이유 6: 업데이트 빈도 낮음

#### 크롤링 주기의 결정 요인

```
검색 엔진이 크롤링 주기를 정하는 기준:

1. 업데이트 빈도
   - 자주 업데이트 → 자주 크롤링
   - 가끔 업데이트 → 가끔 크롤링

2. 사이트 중요도
   - 트래픽 많음 → 자주 크롤링
   - 트래픽 적음 → 가끔 크롤링

3. 콘텐츠 품질
   - 고품질 → 자주 크롤링
   - 저품질 → 가끔 크롤링
```

#### 사이트별 크롤링 주기

```
뉴스 사이트:
- 업데이트: 하루 수십~수백 개 기사
- 트래픽: 매우 높음
- 크롤링 주기: 몇 시간마다
- 최신 데이터: 실시간 반영

정부 사이트:
- 업데이트: 주/월 단위
- 트래픽: 낮음
- 크롤링 주기: 주/월 단위
- 최신 데이터: 지연 반영

결과:
- 뉴스: 검색하면 최신 기사 즉시 나옴 ✅
- 정부: 검색해도 오래된 자료만 나옴 ❌
```

### 이유 7: 검색 키워드 매칭 문제

#### 정부 사이트의 검색 특성

```
일반 웹사이트 (뉴스, 블로그):
- 자연어 친화적
- "조은희 의원이 발의한 법안" → 잘 검색됨
- 관련 키워드 풍부
- 시맨틱 검색 잘 됨

정부 사이트:
- 공식 용어만 사용
- "의안번호 2112345" → 검색됨
- "조은희 법안" → 안 검색됨
- 정확한 키워드 필수
```

#### 예시

```
국회 의안정보시스템 검색:

✅ 잘 되는 검색어:
- "의안번호 2112345"
- "지방자치법 전부개정법률안"
- "제21대국회(의안번호 2112345)"

❌ 안 되는 검색어:
- "조은희 의원 법안"
- "조은희가 만든 법"
- "지방자치 관련 법안"

이유:
→ 데이터베이스가 정형화됨
→ 자연어 검색 지원 안 함
→ 검색 엔진도 마찬가지로 매칭 실패
```

#### Perplexity의 문제

```
Perplexity 사용자:
"조은희 의원이 발의한 법안을 찾아줘"

Perplexity 내부 처리:
1. 자연어를 검색 쿼리로 변환
2. "조은희 의원 법안" → 검색 엔진에 전달
3. 검색 엔진: 뉴스 기사는 찾음 ✅
4. 검색 엔진: 정부 사이트는 못 찾음 ❌

결과:
→ 뉴스 기사만 반환
→ 공식 의안정보 누락
```
