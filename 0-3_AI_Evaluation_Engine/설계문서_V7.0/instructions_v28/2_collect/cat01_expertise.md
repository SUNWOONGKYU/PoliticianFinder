# Category 01: Expertise (전문성) - 수집 지침서 V28

---

## 1. 카테고리 정의

### 한글 정의
직무 수행에 필요한 지식, 기술, 경험의 보유 수준

### 영문 정의
The level of knowledge, skills, and experience required to perform duties effectively

### 학술 근거
- **Stoker (2024)**: Competence의 핵심, "필요한 지식과 기술 보유"
- **Slough (2024)**: 4차원 능력의 기반
- **Volden & Wiseman (2024)**: LES 프레임워크의 전문성 측정
- **LEVER (2024)**: 입법 효과성의 핵심 요소

## 2. 평가 범위

### [O] 전문성에 해당
- 학력, 학위 (고등학교, 대학교, 대학원)
- 전문 자격증 (변호사, 의사, 회계사 등)
- 경력 기간 및 경력 수준
- 직무 관련 전문지식
- 정책 기획 능력
- 예산/재정 이해도
- 학술 활동 (논문, 저서, 강연)
- 법안 발의 실적

### [X] 전문성 아님 (다른 카테고리)
- 조직 관리/위기 대응 → 카테고리 2 (리더십)
- 혁신 아이디어/미래 전략 → 카테고리 3 (비전)
- 부패/비리 → 카테고리 4 (청렴성)
- 약속 이행 → 카테고리 6 (책임성)

### 🚨 [X] 절대 수집 금지 (청렴성/윤리성 이슈!)
**아래 주제는 Expertise가 아님! 해당 카테고리에서만 수집!**

| 주제 | 올바른 카테고리 |
|------|----------------|
| 정치자금 수수 의혹 | Integrity (청렴성) |
| 재산 형성 의혹 | Integrity (청렴성) |
| 인사청문회 재산 논란 | Integrity (청렴성) |
| 가족 비위/표절 | Ethics (윤리성) |

```
❌ 잘못된 예:
Expertise에 "인사청문회 재산 논란" 수집

✅ 올바른 예:
Integrity에 "인사청문회 재산 논란" 수집
```

## 8. 주의사항 (Pitfalls)

1. **학벌 과대평가 금지**: 명문대 출신이라도 경력이 짧으면 객관적으로 기록
2. **경력 허수 구별**: 명목상 경력 vs. 실질적 경력 구분
3. **무관 자격증 제외**: 정치와 무관한 자격증은 수집하지 않음
4. **시대 고려**: 과거 교육 환경 고려 (예: 고졸도 당시엔 평균 이상)
5. **학력 위조 확인**: 의혹과 확정 판결 구분하여 기록

## 8-1. 중복 수집 금지 (CRITICAL!)

### 핵심 규칙
**동일한 사건/주제는 반드시 1개만 수집!**

### 금지 사항
- 같은 사건을 다른 날짜로 반복 수집 ❌
- 같은 사건을 다른 언론사로 반복 수집 ❌
- 같은 주제를 다른 표현으로 반복 수집 ❌

### 예시: 중복 수집 (잘못된 예)
```
[X] "서울대 법학과 졸업" (출처 A)
[X] "서울대학교 법학과 학사" (출처 B)
[X] "서울대 법대 졸업" (출처 C)
→ 모두 같은 학력! 1개만 수집해야 함
```

### 예시: 올바른 수집
```
[O] "서울대 법학과 졸업 (1985)" - 1개만!
[O] "변호사 자격증 취득 (1988)" - 별개 항목이므로 OK
[O] "행정고시 합격 (1987)" - 별개 항목이므로 OK
```

### 동일 사건 판단 기준
- 같은 학력/자격증/경력 정보
- 같은 법안 발의 건
- 후속 보도, 재조명 = 원래 사건과 동일

### 체크 방법
수집 전 스스로 확인:
1. "이 주제를 이미 수집했나?"
2. "비슷한 제목의 항목이 있나?"
3. "같은 사건의 다른 보도인가?"

**중복 수집 시 전체 응답 무효 처리됨!**

## 9. JSON 출력 형식

```json
{
  "items": [
    {
      "item_num": 1,
      "data_title": "간결한 제목 (20자 이내)",
      "data_content": "구체적 내용 (사실 기반, 100-300자)",
      "data_source": "구체적 출처명",
      "source_url": "https://...",
      "source_type": "OFFICIAL",
      "data_date": "2025-06-15"
    }
  ]
}
```

### 필드 설명
| 필드 | 설명 | 필수 |
|------|------|------|
| item_num | 항목 번호 (1~50) | O |
| data_title | 간결한 제목 (20자 이내) | O |
| data_content | 구체적 내용 (100-300자) | O |
| data_source | 출처명 (기관/언론사명) | O |
| source_url | 실제 존재하는 URL | O |
| source_type | "OFFICIAL" 또는 "PUBLIC" | O |
| data_date | YYYY-MM-DD 형식 | O |

### 주의
- **rating 필드 포함하지 마세요!** 수집 단계에서는 평가 없음
- source_type은 반드시 대문자
