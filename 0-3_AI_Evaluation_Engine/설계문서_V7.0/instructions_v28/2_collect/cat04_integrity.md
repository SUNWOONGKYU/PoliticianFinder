## 1. 카테고리 정의

## 8. 주의사항 (Pitfalls)

1. **의혹 vs 확정 구분**: 수사 중 vs 유죄 확정 명확히 구분
2. **정치적 공격 구분**: 정쟁성 공격 vs 실제 비리 구별
3. **금액 규모 기록**: 비리 금액 규모 함께 기록
4. **시기 명시**: 사건 발생 시기 명확히 기록
5. **결과 추적**: 수사/재판 결과 함께 기록

## 8-1. 중복 수집 금지 (CRITICAL!)

### 핵심 규칙
**동일한 사건/주제는 반드시 1개만 수집!**

### 금지 사항
- 같은 사건을 다른 날짜로 반복 수집 ❌
- 같은 사건을 다른 언론사로 반복 수집 ❌
- 같은 주제를 다른 표현으로 반복 수집 ❌

### 예시: 중복 수집 (잘못된 예)
```
[X] "손길승 불법자금 수수 논란 재점화" (2024-09-10)
[X] "손길승 회장 불법 자금 수수" (2022-03-01)
[X] "손길승 불법 정치자금 사건" (2025-01-05)
→ 모두 같은 사건! 1개만 수집해야 함
```

### 예시: 올바른 수집
```
[O] "손길승 불법자금 수수 (2002년 유죄 확정)" - 1개만!
[O] "인사청문회 재산 형성 의혹" - 별개 사건이므로 OK
[O] "아들 표절 의혹" - 별개 사건이므로 OK
```

### 동일 사건 판단 기준
- 같은 인물이 관련된 같은 유형의 비위
- 같은 시기에 발생한 같은 사건
- 후속 보도, 재조명, 재점화 = 원래 사건과 동일

### 체크 방법
수집 전 스스로 확인:
1. "이 주제를 이미 수집했나?"
2. "비슷한 제목의 항목이 있나?"
3. "같은 사건의 다른 보도인가?"

**중복 수집 시 전체 응답 무효 처리됨!**

## 8-2. URL 규칙 (CRITICAL!)

### 핵심 규칙
**출처 유형에 따라 URL 필수 여부가 다름!**

### SNS (URL 불필요)
다음 플랫폼은 source_url 빈 값 허용:
- 트위터/X
- 페이스북
- 유튜브
- 인스타그램

### 웹사이트 (URL 필수!)
다음 출처는 **반드시 실제 URL 제공**:
- 언론사 기사 (조선, 중앙, 한겨레 등)
- 위키 (나무위키, 위키피디아)
- 시민단체 보고서
- 커뮤니티 게시물
- 기타 모든 웹 기반 출처

### 금지 사항
- ❌ URL을 모르면서 데이터 수집 금지
- ❌ 가짜 URL 생성 절대 금지
  - `/articles/123456` 같은 패턴
  - `example.com` 사용
  - 존재하지 않는 URL 조작

### 예시
```
[O] SNS: source_url = "" (빈 값 OK)
[O] 언론: source_url = "https://www.chosun.com/..." (실제 URL)
[X] 언론: source_url = "" (URL 없으면 수집 금지!)
[X] 가짜: source_url = "https://news.com/articles/123" (가짜 URL!)
```

**URL 규칙 위반 시 전체 응답 무효 처리됨!**
