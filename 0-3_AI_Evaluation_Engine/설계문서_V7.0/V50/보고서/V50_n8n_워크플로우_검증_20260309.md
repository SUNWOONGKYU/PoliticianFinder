## 2. 발견 및 수정된 버그

### 버그 1 (Critical) — P5: Generate HTML Report 크래시
**노드**: `v50-p5-gen-report`
**원인**: `Supabase: Save Scores`가 `Prefer: return=minimal`로 응답 → `{}` 반환
→ 다음 노드 P5에서 `const p = $input.first().json` → p = `{}`
→ `Object.entries(p.category_scores)` → TypeError: Cannot convert undefined

**수정**:
```javascript
// Before (CRASH)
const p = $input.first().json;

// After (FIXED)
const p = $('P4: Calculate Scores').first().json;
```

---

### 버그 2 (High) — Status: done 노드 report_path NULL 저장
**노드**: `v50-status-done`
**원인**: 직전 노드 `Supabase Storage: Upload Report`의 응답 스키마는 `{ Key: "..." }`.
`$json.report_path`를 참조했으나 Storage 응답에 해당 필드 없음 → `undefined` → DB에 NULL 저장.

**수정**:
```javascript
// Before (NULL 저장)
report_path: $json.report_path

// After (FIXED)
report_path: $('P5: Generate HTML Report').first().json.report_path
```

## 3. 전체 드라이런 추적 경로

```
Webhook → Validate Input → P0:Register Politician
→ Status:collecting → Generate Collection Prompts
→ [Collection Batch Loop × 10라운드]
    Route:Gemini? → Gemini API:Collect  ┐
    Route:Naver?  → Naver API:Collect   ├→ Merge 3 Channels → Parse → Save Collected
                 → Grok-X API:Collect  ┘
    → Continue Loop → [loop back]
→ Gate1:Collection Count → Check Gate1 → IF Gate1 Pass
→ P2:Validate → Status:validating
→ Check Balance → IF Balance OK
    (false) → Recollect Gemini → [loop back to Check Balance]  ← 최대 4라운드
    (true)  → Status:evaluating → Prepare Eval Prompts
→ [Category Eval Loop × 40라운드 (10 cats × 4 batch_indices)]
    Fetch Batch Data → IF Data Exists?
      (false) → [skip, loop back]
      (true)  → Build Eval Prompt
               → Claude / Gemini / ChatGPT / Grok API: Evaluate (4병렬)
               → Merge 4 AI Evaluations → Parse 4 AI Responses
               → Supabase: Save Evals → [loop back]
→ Status:scoring → P4:Calculate Scores
→ Supabase:Save Scores
→ P5:Generate HTML Report  ← Bug1 수정지점
→ Supabase Storage:Upload Report
→ Status:done               ← Bug2 수정지점
→ Final Summary ✅
```
