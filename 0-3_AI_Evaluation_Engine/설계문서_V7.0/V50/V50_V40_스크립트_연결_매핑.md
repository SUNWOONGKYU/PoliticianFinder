# V50 ← V40 스크립트 연결 매핑

**작성일**: 2026-03-08
**목적**: V40 CLI/Python 스크립트 → V50 n8n HTTP Request/Code Node 1:1 대응표

---

## 핵심 원칙

| 구분 | V40 | V50 |
|------|-----|-----|
| 실행 방식 | CLI subprocess / Python 직접 실행 | n8n 워크플로우 노드 |
| 인증 | 구독 계정 (Claude/Gemini CLI) + API Key | **전부 API Key** |
| 스케일 | 1명씩 수동 | 분대원(Subagent) → soldier_v50.py → n8n webhook |
| 비용 | Claude $0, Gemini $0 (CLI 구독) | 전체 ~$0.41/명 |

**V40의 로직(알고리즘)은 그대로 V50에 이식. 실행 방식만 변경.**

## 전체 n8n 노드 구성 요약

```
[Webhook 수신: soldier_v50.py → POST /webhook/v50-start]
         ↓
[Phase 0] Supabase Node: 정치인 등록 확인 / INSERT
         ↓
[Phase 1] HTTP Request × 3 (병렬):
  ├─ Gemini 2.0 Flash-Lite API     ← collect_gemini_subprocess.py 대체 (40%, 48개)
  ├─ Grok xAI Live Search API      ← (V50 정규 채널, 10%, 12개)
  └─ Naver Cloud Platform API      ← collect_naver_v40_final.py 대체 (50%, 60개)
         ↓
[Phase 2] Code Node: 중복제거·기간필터  ← validate_v40_fixed.py 로직 이식
         ↓
[Phase 2-2] Code Node: 균형 조정       ← adjust_v40_data.py 로직 이식
  └─ 부족 시 Phase 1 노드 재호출 (최대 4회)
         ↓
[Phase 3] HTTP Request × 4 (병렬):
  ├─ Anthropic API (claude-haiku-4-5) ← claude_eval_helper.py 대체
  ├─ OpenAI API (gpt-4o-mini)         ← codex_eval_helper.py 대체
  ├─ Gemini API (2.0-flash-lite)      ← evaluate_gemini_subprocess.py 대체
  └─ xAI API (grok-3-mini)            ← grok_eval_helper.py 대체
         ↓
[Phase 3-1] Code Node: 평가 결과 검증 (V50 신규)
         ↓
```
