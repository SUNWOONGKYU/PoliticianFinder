- **도구**: Grok API (xAI Live Search)

## 5. 소대편제 아키텍처 (소대 1개 = 세션 1개)

```
지휘관 (사용자) — /evaluate-platoon-v50 실행, 3가지 질문에 답변
    ↓
소대장 (Claude Code 세션 1개 = 소대 1개) — TeamCreate, 배치 관리
    ↓
분대장 × 5 (Teammate, Sonnet) — 분대원 5명 스폰 + 완료 보고
    ↓
분대원 × 25 (Subagent, Haiku) — soldier_v50.py 실행 (webhook + polling)
    ↓
n8n Sub-workflow — Phase 0-5 실제 실행 (수집·검증·평가·점수·[PO승인]·보고서)
    ↓
4 AI 평가 API — Claude Haiku / Gemini Flash-Lite / gpt-4o-mini / grok-3-mini
```

### 처리 속도 (소대 1개 기준, 정치인 1명)

| 단계 | 시간 |
|------|------|
| Phase 1 수집 (3채널 API) | 실측 필요 |
| Phase 2 검증·조정 | 실측 필요 |
| Phase 3 평가 (4AI 병렬) + 3-1 검증 | 실측 필요 |
| Phase 4 점수 계산 | 실측 필요 |
| Phase 4.5 PO 승인 게이트 | CCI 결과 보고 후 PO 승인 대기 (인간 개입 필수) |
| Phase 5 보고서 생성 | 실측 필요 |
| n8n 오버헤드 | 실측 필요 |
| **1명 완료** | **실측 후 확정** |

**편제별 처리량 (소대 1개):**

| 편제 (M분대 × N명) | 소대 총원/배치 | 배치 수 (P명 처리 시) | 소요 시간 |
|------------------|------------|-------------------|---------|
| 3분대 × 4명 (최소) | 12명 | ceil(P / 12) 배치 | 실측 후 확정 |
| 5분대 × 5명 (기본) | 25명 | ceil(P / 25) 배치 | 실측 후 확정 |
| 5분대 × 7명 (빠름) | 35명 | ceil(P / 35) 배치 | 실측 후 확정 |
| 5분대 × 10명 (최대) | 50명 | ceil(P / 50) 배치 | 실측 후 확정 |

```
편제는 스킬 시작 시 지휘관이 선택:
  분대 수: 3-5개 (기본 5개)
  분대원 수: 4-10명 (기본 5명, Claude Code 한도 10명)
  처리 대상(P): 지휘관이 결정 (규모 제한 없음)

소요 시간 = T(실측) × ceil(P / M×N) / S (S = 병렬 소대 수)
```

### 배치 완료 후 루프

```
25명 배치 완료 → 소대장이 지휘관에게 3가지 질문:
  ① 실패 재실행? → 실패 정치인 재배치
  ② 추가 임무? → 다음 25명 배치 시작
  ③ 해산? → TeamDelete → 소대 종료
```

## 9. 실시간 모니터링 대시보드

### 필요성

다수 정치인 동시 처리 시 터미널 SQL 쿼리로는 현황 파악 불가.
다수 소대(세션)가 동시 운영되므로, **실시간 웹 대시보드**가 필수.

### 기술 스택

| 항목 | 선택 |
|------|------|
| 프론트엔드 | Next.js (기존 프로젝트 동일) |
| 실시간 갱신 | Supabase Realtime (WebSocket 구독) |
| 데이터 소스 | `politicians` 테이블 `processing_status` 컬럼 |
| 배포 | Vercel (기존 인프라) |

### Supabase Realtime 구독

```typescript
// 실시간 processing_status 변경 감지
supabase
  .channel('v50-monitor')
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'politicians',
    filter: 'eval_week=eq.2026-W10'
  }, (payload) => {
    updateDashboard(payload.new)
  })
  .subscribe()
```

### 집계 쿼리

```sql
-- 전체 현황 집계 (대시보드 초기 로드)
SELECT processing_status, COUNT(*) as cnt
FROM politicians
WHERE eval_week = '2026-W10'
GROUP BY processing_status
ORDER BY processing_status;
```
