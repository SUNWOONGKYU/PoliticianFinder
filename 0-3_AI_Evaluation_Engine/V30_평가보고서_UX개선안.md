## 1. 현재 UX 분석

### Heuristic Evaluation

#### 1.1 발견성 (Discoverability): 70/100점

**강점**:
- 명확한 섹션 구분 (9개 메인 섹션)
- 목차 없어도 섹션 제목만으로 내용 파악 가능
- 번호 매기기 (1, 2, 3...)로 순서 명확

**약점**:
- 목차(TOC) 없음 - 1,349줄에서 원하는 섹션 찾기 어려움
- 섹션 간 계층 구조 불명확 (h2, h3, h4 혼재)
- 페이지 점프 링크 없음

#### 1.2 학습 용이성 (Learnability): 85/100점

**강점**:
- "한눈에 보는 평가 요약" (섹션 2) - 3분 안에 핵심 파악 가능
- 반복적 구조 (강점 5개, 약점 3개, 카테고리 10개 모두 동일 형식)
- 예시와 함께 설명 (AI 평가 근거 직접 인용)

**약점**:
- 초심자가 이해하기 어려운 용어 (OFFICIAL, PUBLIC, Rating, X 등)
- 용어 설명 없음 (섹션 9까지 가야 시스템 설명)

#### 1.3 효율성 (Efficiency): 60/100점

**강점**:
- 표를 활용한 빠른 정보 스캔 가능
- 시각적 구분선 (---)으로 섹션 구분

**약점**:
- 긴 텍스트 블록 (문단이 10줄 이상인 경우 다수)
- 중요 정보 강조 부족 (굵게, 색상, 박스 등 미흡)
- 스크롤 피로도 높음 (1,349줄)

#### 1.4 만족도 (Satisfaction): 75/100점

**강점**:
- 긍정적 마무리 (섹션 10 "의원님의 성공을 응원합니다!")
- 실행 가능한 개선 방안 제시
- 체크리스트로 다음 단계 명확

**약점**:
- 감정적 공감 부족 (데이터 중심, 사람 중심 아님)
- 시각적 즐거움 부족 (텍스트만 있음, 차트/그래프 없음)

```markdown
┌─────────────────────────────────────────────┐

│ Grok   │  127개   │  31개   │ 84점 ⭐      │

└─────────────────────────────────────────────┘
```

**개선 효과**:
- 박스로 구분하여 시각적 분리

## 8. 사용자 테스트 계획

### 8.1 테스트 대상

1. **정치인 본인** (Primary User)
   - 강점/약점 빠르게 파악 가능한가?
   - 실행 가이드가 유용한가?

2. **보좌관** (Secondary User)
   - 데이터 출처 신뢰할 수 있는가?
   - 카테고리별 상세 분석이 도움 되는가?

3. **일반 시민** (Tertiary User)
   - 정치인을 이해하는 데 도움 되는가?
   - 용어가 이해하기 쉬운가?

### 8.2 테스트 시나리오

**시나리오 1**: 3분 안에 핵심 파악
- 목표: Executive Summary 읽고 핵심 파악
- 측정: 정확도, 소요 시간

**시나리오 2**: 약점 개선 방안 찾기
- 목표: 청렴성 약점 개선 방안 찾기
- 측정: 찾기까지 소요 시간, 만족도

**시나리오 3**: 특정 카테고리 상세 분석
- 목표: "비전" 카테고리 상세 평가 읽기
- 측정: 정보 충분성, 이해도

### 8.3 평가 지표

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| Task Success Rate | 작업 완료율 | 90% 이상 |
| Time on Task | 작업 소요 시간 | 목표 시간 내 |
| Satisfaction (SUS) | 설문 (1-5점) | 4점 이상 |
| Findability | 원하는 정보 찾기 시간 | 30초 이내 |
